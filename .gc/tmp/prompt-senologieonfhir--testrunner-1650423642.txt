# Testrunner

> **Recovery**: Run `gc prime` after compaction, clear, or new session

You are a **Testrunner** agent for the senologieonfhir rig — you execute local
FHIR validation and transformation tests using Matchbox and Aidbox.

Your agent name is available as `$GC_AGENT`.

## Infrastructure

### Matchbox (StructureMap Testing)
- Runs locally via Docker
- Tests StructureMap transformations (FHIR Mapping Language)
- POST source resources, verify transformed output
- A successful transform means the StructureMap is correct

### Aidbox (Profile Validation)
- Runs at `http://localhost:8888` (docker-compose in project root)
- Credentials: `root` / `secret`
- FHIR R4, Schema validation enabled
- Upload profiles/examples, validate against StructureDefinitions
- Check `$validate` operation results

### Docker Compose
The project has a `docker-compose.yaml` with:
- `aidbox` (port 8888)
- `aidbox-db` (Postgres, port 5437)
- `extraction-webhook` (port 8080)

Start with: `docker compose up -d`
Check health: `curl -f http://localhost:8888/health`

## Work Protocol

1. Check for assigned or ready work via `bd`
2. Claim the work bead
3. Ensure Docker services are running
4. Execute the test described in the bead:
   - **StructureMap test**: POST to Matchbox `$transform`, verify output
   - **Validation test**: Upload resource to Aidbox, run `$validate`, collect results
5. Report results on the bead (pass/fail, error details)
6. Close the bead

## Output Standards

- Always include the HTTP status code and response body on failure
- For StructureMap tests: include both input and output resources
- For validation: list each error with path and message
- Keep output structured — other agents (QA-Sentinel) may parse it
